Machine Learning

Home robotics can be heavily reliant on machine learning algorithms, as it is often easier and more practical to have a robot learn to do the tasks, rather than to hard code every possible scenario into it.

There are two main methods of machine learning methods, supervised and unsupervised. Each method has its own merits and usage.


Supervised (1)


Machines that are based on supervised learning algorithms require labelled training data. The algorithm will learn to map the input to output with a function, y = f(x), where y is the output variable, and x is the input variable. The goal of this algorithm is to predict the output of new events based on past events.

To solve a problem using supervised learning, one must do the following:
    1. Determine what the training examples will consist, for example if you want to predict housing prices, you’ll need to decide on factors such as whether the property is commercial, residential, or industrial, what country it is in, etc.
    2. Gather the training data, in this case it’ll have to include determinants such as apartment size, age, location, with the corresponding output, price.
    3. Typically, the input data is transformed into a vector, which contains several features that describe the object. However, the number of features should not be too great, because of the curse of dimensionality, but enough relevant information for an accurate output prediction.
    4. Select a structure for the learned function and algorithm. For example, support vector machine.
    5. Run the learning algorithm on the training set to test it, some algorithms require control parameters to be set.
    6. Evaluate the accuracy of the learned function using a set of data that is not the training set.

There are four major downsides to supervised learning.
Curse of dimensionality


It refers to a phenomenon that occurs when analyzing and organizing high-dimensional spaces. As the dimension increases, the amount of data required to support the result grows exponentially. Some of the features of the data might have no correlation with the output, but due to the way supervised learning works, it’ll still take into account irrelevant data for its predictions. This ties into Hughes phenomenon, which states an inverse relationship between predictive power and dimensionality.


Bias-Variance tradeoff


If an algorithm has low-bias, then it’ll fit well with the data, but leading to it fitting to each set of data differently, and hence high variance. On the other hand, an algorithm with low variance will not have an accurate output prediction.


Amount of training data available


There might not be enough training data available for the user to use, and functions of high complexity will require a huge amount of training data.


Error in input data


In huge amounts of training data, there are bound to be some errors, let it be random or systematic errors. These errors will result in outliers the mapping function, and can corrupt the training data as the algorithm will attempt to fit the incorrect data into its function.


Unsupervised (1)


Unlike supervised machine learning algorithms, unsupervised machine learning algorithms use unlabelled training data. One of the main distinctions between unsupervised and the other two types of machine learning is that there is no accurate evaluation on the output, as the input has no actual meaning behind it (label)

There are several types of unsupervised machine learning algorithms, including clustering and neural networks.


Clustering


As mentioned previously, unsupervised algorithms are given unlabelled data. The task of clustering algorithms is to split the data into several clusters, each cluster having a certain trait. There are many different clustering algorithms, as seen in this Wikipedia List (https://en.wikipedia.org/wiki/List_of_algorithms#Statistics).

One of the prominent types of clustering algorithms is known as hierarchical clustering, which clusters its data based on the ‘distance’ between each set of dat. The distance being the result of a metric being applied to two points on a graph of two or more dimensions. These metrics include the Euclidean distance, Manhattan distance etc.

Another common type of clustering models is known as k-means clustering. It will partition the data into k clusters. At the centre of each cluster is a centroid. At first, centroids are randomly placed, and each data point is mapped to the closest cluster. The clusters then find its new centroid based on an average of its assigned points. These two steps are repeated until the centroids no longer change.

One of the criticisms of k-means clustering is that it requires an input of k, and a poor choice of k will result in poor output.

Neural Networks(3)(4)


Neural networks are a collection of artificial neural units, with connectives between each neuron and many others. The goal of neural networks is to simulate how a human would solve a problem, and mainly used to recongize patterns from a series of data.

Neurons within a neural network are arranged in layers, there are at least three layers, including input, and output layers, as well as one or more hidden layers. The 'synapses' which connect the neurons together each carry a weight, and the total weight of all the synapses connected to a neuron determines its state. The state of a neuron is either activated, or not. If it is activated, then it will continue to affect the connected neurons on the next layer, until eventually the output layer is reached, and the activated neurons represents the output.
